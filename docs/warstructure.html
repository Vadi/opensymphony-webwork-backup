WebWork 2 WAR Structure

Example Directory Structure

The following is one example of a web application directory structure that can be used to configure WebWork 2 in your application.

  • /webapp/WEB-INF
  • /webapp/WEB-INF/web.xml
  • /webapp/WEB-INF/webwork.tld
  • /webapp/WEB-INF/classes
  • /webapp/WEB-INF/classes/xwork.xml
  • /webapp/WEB-INF/lib

For more information about the web.xml file, see Configuring web.xml.

WebWork 2 has a number of required and optional libraries that must be included in the WEB-INF/lib folder for certain features of the framework to work correctly.

Required libraries:

  • commons-logging.jar (this is the logging library used in webwork2)
  • ognl-2.5.1.jar (this is the OGNL library used for webwork2's expression language)
  • oscore-2.2.1.jar (contains core functionality that is reused in webwork2)
  • xwork-1.0-beta.jar
  • webwork-2.0-beta.jar

Optional libraries:

  • cos-multipart.jar (only required if you need cos multipart support)
  • pell-multipart.jar (only required if you need pell multipart support)
  • velocity-dep-1.3.jar (only required if you use the webwork2 tag libraries)

Note that commons-logging will pick a specific log implementation only if it finds the correspondent classes on the classpath. So, if you want commons-logging to use Log4J as the actual log implementation, you need to put the log4j.jar file in the WEB-INF/lib folder as well.
