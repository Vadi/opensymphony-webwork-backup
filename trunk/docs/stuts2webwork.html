The Struts to WebWork 2 bridge

Here's a thought. WebWork 2 could run a Struts application with only a little modification. If there's interest, I'll write this tool. My thought is the simple path migration would look like this:

  1. download the struts 2 webwork 2 bridge jar that comes with replacement struts-*.tld files
  2. provide instructions on configuring web.xml
  3. provide instructions on automatically converting struts-config.xml to xwork.xml
  4. drop the bridge jar file into WEB/lib
  5. you're done!

The bridge jar file should contain our implementation of the Struts Action, ActionForm, and ActionError classes as well as an implementation for some of the key Struts html tags.

This would give developers familiar with Struts a way to ease themselves into WebWork 2 using an application they already know.
