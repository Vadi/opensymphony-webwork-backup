Checkbox Tag

Remarks

This tag creates an HTML input control of type checkbox. If the 'label' attribute is specified, its value will be displayed to the right of the checkbox.

Details
Class: com.opensymphony.webwork.views.jsp.ui.CheckboxTag
Syntax: <ui:checkbox ... />
Tag Body: empty
Template: checkbox.vm
Theme: /decorators/xhtml/
Attributes
Attribute Description Required
label  The text to display with this component. The text will be displayed to the right of the checkbox. true
name  The name of the component. This value should be implemented in your action using the JavaBeans notation to allow WebWork to automatically set and get this value. true
template  A user specified template to use. This value will default to checkbox.vm. false
theme  A user specified theme to use. This value will default to /decorators/xhtml/. false
value  The default value for the checkbox. If this value is set to true, the checkbox will be checked by default. false

- Inheirited from superclass.

Changes from WebWork 1

In WebWork 1, the value of the checkbox was set using the 'fieldValue' attribute. In WebWork 2, this value is set using the 'value' attribute. The following attributes have not been implemented in WebWork 2: id, labelposition, disabled, tabindex, and onchange.

Example

In this example, a checked checkbox will be displayed. When the form is submitted to an action, the setAdditionalInfo(boolean additionalInfo) method will be called.

Jsp code:

<ui:checkbox name="additionalInfo"
             label="Check here to receive additional information. ..."
             value="true" />

Output using the default templates:

<td colspan="2">
<table width="100%" >
<tr>
  <td valign="top" align="right"><input type="checkbox" name="additionalInfo" checked="checked"/></td>
  <td>Check here to receive additional information. ...</td>
</tr>
</table>
</td>

Component Tag

Remarks

This tag allows you to incorporate custom UI templates for rendering. See Creating Custom Components.

Details
Class: com.opensymphony.webwork.views.jsp.ui.ComponentTag
Syntax: <ui:component ... />
Tag Body: empty
Template: empty.vm
Theme: /decorators/xhtml/
Attributes
Attribute Description Required
label  This attribute is defined to allow custom templates to access its value. This value is typically used to give a visual description next to a component. false
name  This attribute is defined to allow custom templates to access its value. This value is typically used in action classes using the JavaBeans notation to allow WebWork to automatically set and get this value. false
template  This value is required and should point to a Velocity template file that describes your component. true
theme  A user specified theme to use. This value will default to /decorators/xhtml/. false
value  This attribute is defined to allow custom templates to access its value. false

- Inheirited from superclass.

Changes from WebWork 1

In WebWork 1, the 'template' attribute pointed to a Jsp file or fragment. In WebWork 2, the 'template' attribute should point to a Velocity template. For more information see Creating Custom Components. The following attributes have not been implemented in WebWork 2: labelposition.

Example

For an example, see Creating Custom Components.

Form Tag

Remarks

This tag creates an HTML form element. The 'action' attribute will be URL encoded if specified.

Details
Class: com.opensymphony.webwork.views.jsp.ui.FormTag
Syntax: <ui:form> ... </ui:form>
Tag Body: Jsp
Template: form.vm, form-close.vm
Theme: /decorators/xhtml/
Attributes
Attribute Description Required
action This attribute specifies a form processing agent, usually an HTTP URI. true
closeTemplate A user specified template to use. This template is responsible for the closing part of the form tag. This value will default to form-closing.vm. false
enctype This attribute specifies the content type used to submit the form to the server (when the attribute 'method' is set to "post"). false
method This attribute specifies which HTTP method will be used to submit the form. Possible (case-insensitive) values are "get" (the default) and "post". false
name  This attribute specifies the value to use in the form's name attribute. false
template  A user specified template to use. This template is responsible for the opening part of the form tag. This value will default to form.vm. false
theme  A user specified theme to use. This value will default to /decorators/xhtml/. false

- Inheirited from superclass.

Changes from WebWork 1

This tag is new in WebWork 2.

Example

In this example, a form is created using a WebWork action class for the 'action' attribute. The form will POST its results when submitted.

Jsp Code:

<ui:form action="SimpleAction.action" method="POST" name="frmLogin">
<table>
  <ui:textfield name="username" label="Username" size="25" />
</table>
</ui:form>

Output using the default templates:

<form name="frmLogin" action="SimpleAction.action" method="POST">

<table>
  <tr>
  <td align="right" valign="top"><span class="label">Username:</span></td>
  <td>
<input type="text" name="username" value="" size="25"/>
  </td>
</tr>
</table>
</form>
