Overview

The following diagram provides a high level illustration of how WebWork 2 processes web requests.

  1. All incoming requests to WebWork are handled by ServletDispatcher
  2. The ServletDispatcher looks up a reference to a new ActionProxy instance
  3. The ActionProxy wraps calls to the Action using the defined Interceptors.
  4. Once all the Interceptors and the Action have been executed the Action call the Result to render the document.
